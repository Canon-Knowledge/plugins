# Verification checklist — run before delivering

Do not deliver the knowledge base until every box is checked, or its gap is
explicitly written into section 11 ("notas para quem for treinar o bot").

## Completeness
- [ ] `/sitemap.xml` checked (or main menu fully enumerated).
- [ ] Every content domain from discovery has a section.
- [ ] **Pagination verified** on every listing page (catalog, locations, FAQ).
- [ ] **Counts reconciled**: captured count compared to the total the site
      declares; any divergence is flagged in section 11.
- [ ] Duplicates across categories merged into a single entity (categories noted).

## Sales fields (per catalog item, where available on the site)
- [ ] Price / conditions captured (or marked "não disponível" + URL).
- [ ] Promotions / bundles / upsell-cross-sell.
- [ ] CTA / conversion path (WhatsApp, buy, quote, checkout).
- [ ] Objection handling (warranty, returns/refund, deadlines, FAQ).
- [ ] Social proof (testimonials, awards, ratings).
- [ ] Urgency/scarcity triggers.
- [ ] USPs / taglines / brand positioning.
- [ ] Tone-of-voice verbatim samples.
- [ ] Lead-capture mechanisms.

## Sourcing & integrity
- [ ] Every fact/price/condition cites its source URL.
- [ ] No invented data — missing fields marked, not fabricated.
- [ ] JS-only / empty pages flagged for manual review, not silently dropped.
- [ ] No authenticated / account / third-party-personal-data pages accessed.

## Verification subagent
- [ ] Sample of 8–12 links across sections validated (resolve + match content).
- [ ] Result reported in document and in chat.

## Delivery & follow-up
- [ ] Primary Markdown produced; extras (`.docx`/`.xlsx`/`.json`) only if asked.
- [ ] Section 11 present (gaps, JS pages, count divergences, refresh cadence).
- [ ] Periodic re-run offered (schedule / scheduled tasks).
